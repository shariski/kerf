# Task 4.4 — Accessibility basics — Design

**Status:** draft → pending user review
**Target PR branch:** `feat/task-4.4-accessibility` (off `main`)
**Scope:** `docs/03-task-breakdown.md §4.4` — keyboard navigation, screen reader labels, WCAG 2.1 AA color contrast, reduced-motion handling. Driven by `CLAUDE.md §B4` which has been partially enforced throughout development; this task formalizes the contract and closes remaining gaps.

## Why

kerf is built for users who rely heavily on keyboards — the product
literally cannot function without keyboard fluency. Failing WCAG 2.1 AA
for keyboard/focus concerns would be tone-deaf for a typing-training
platform. `CLAUDE.md §B4` has been followed in ad-hoc form:
`prefers-reduced-motion` guards on animations, aria attributes on
interactive elements, semantic HTML. No formal audit has run, though,
and a lightning scan surfaced concrete gaps (global focus-ring missing,
some `outline: none` without replacement, contrast never formally
verified). Task 4.4 closes those gaps and locks the contract in a
documented, reproducible form.

## User decisions locked in (2026-04-22)

- **Scope depth:** full audit (option C from brainstorming) — install
  `@axe-core/playwright`, enumerate every route + key inner state, fix
  all critical + serious violations, zero-violation baseline committed.
- **WCAG target:** 2.1 AA. Not AAA. Not deep real-screen-reader QA (that
  is post-beta).
- **Automation:** local `npm run test:a11y` script only; **no CI
  integration** — `CLAUDE.md §B9` makes CI out of scope for Phase A.
- **Single PR** structured in three waves: tooling + baseline,
  critical/serious fixes, docs + verification.

## Non-goals (explicitly out of scope)

- CI integration (axe as a blocking check on PRs) — no CI exists yet per `§B9`.
- WCAG 2.1 AAA compliance.
- Real screen-reader QA with NVDA/JAWS/VoiceOver — automated axe only.
- Internationalization / RTL support.
- Deep Recharts a11y (chart titles + descriptions yes; full keyboard navigation of chart data points no).
- Keyboard-trap-free modals beyond the existing pause overlay. No new modals in this PR.
- Landmark-based "skip to dashboard section" navigation — vim scroll is enough for Phase A.

## Design

### Wave 1 — Tooling + baseline audit

**Dependencies to install (devDependencies):**

- `@playwright/test` — test runner + browsers
- `axe-core` — the engine
- `@axe-core/playwright` — the Playwright integration

No Playwright browsers need to be committed; `npm install` will pull
them (or the developer runs `npx playwright install` once locally).

**New files:**

- `playwright.config.ts` — minimal config: web-server points at
  `npm run dev` on port 3000, one browser project (Chromium), no
  per-project base URL overrides.
- `tests/a11y.spec.ts` — the single a11y test suite.

**Routes tested by the audit** (every state exercisable without typing a live session):

| Route | State |
|-------|-------|
| `/` | Returning-user (assumes seeded dev DB has sessions) |
| `/login` | Logged-out |
| `/onboarding` | Step 1 |
| `/practice` | Pre-session |
| `/practice/drill` | Pre-drill target picker |
| `/practice` with Esc pressed | Pause overlay |
| `/dashboard` | Full dashboard |
| `/keyboards` | Profile grid |
| `/` at 375×812 viewport | Mobile gate |

States that **require** typing a session (active typing, post-session
summary, drill active/post) are excluded from the automated sweep —
they are behind a user interaction too fragile to script. Manual check
suffices for Phase A; note in `docs/a11y.md` as a known deferral.

**Axe rules:** run with tags `['wcag2a', 'wcag2aa', 'wcag21a', 'wcag21aa']`.

**Baseline commit policy:** commit the axe spec even if it initially
reports violations. Wave 2 fixes follow; Wave 3 confirms zero critical
+ zero serious.

### Wave 2 — Fix critical + serious violations

**Commit 2a — Keyboard focus baseline**

- Global `:focus-visible` ring in `src/styles.css` near the top of the file:
  ```css
  :focus-visible {
    outline: 2px solid var(--color-kerf-amber-base);
    outline-offset: 2px;
    border-radius: 2px;
  }
  ```
- Audit all existing `outline: none` declarations in `styles.css` (currently ~4):
  - `.kb-key-clickable:focus` — already has a sibling `:focus-visible > .kb-key-face` rule; verify it renders visibly
  - `.kerf-drill-target-input:focus` — add a `:focus-visible` variant with amber border or box-shadow
  - `.kerf-kb-switcher-item:focus-visible` / `.kerf-kb-switcher-manage:focus-visible` — already use `background` instead of outline; confirm AA contrast ratio
- Skip-to-main-content link at the top of `__root.tsx`:
  ```tsx
  <a href="#main-content" className="kerf-skip-link">Skip to main content</a>
  ```
  Visually hidden until `:focus` moves it into view. Every route component gets `id="main-content"` on its top-level `<main>`.

**Commit 2b — Heading hierarchy + landmarks**

- Every route renders exactly one `<h1>`, no level skips.
- Verify all routes wrap their body in a `<main>` element (dashboard, practice, keyboards already do). The `AppNav` is in a `<header role="banner">`.
- Add the `id="main-content"` to each route's `<main>` (paired with the skip link from 2a).
- Confirm `<html lang="en">` is set — it already is in `__root.tsx`.

**Commit 2c — Screen reader labels**

- AppNav settings-cog button — add `aria-label="Settings"` (currently `disabled` placeholder; keep it labelled).
- AppNav avatar button — if unlabelled, add `aria-label="User menu"`.
- Every decorative emoji / glyph (`⌨`, `⊞`, `⏎`, `▼`, `●`, `◎`, `◷`, `⬌`, `↵`, `→`, etc.) gets `aria-hidden` where it isn't already. Axe flags untitled icons so this is the triage pass.
- `LiveWpm` display — add `aria-live="off"` explicitly (the counter updates too rapidly to be useful as announcements). Add `aria-label="Words per minute"` to the element so a screen reader user can find it on demand.
- Any `<img>` missing `alt` gets either a real alt or `alt=""`. (The `KeyboardSVG` components already use `role="img"` + `aria-label`.)

**Commit 2d — Contrast fixes**

- Run axe with `only: { values: ['color-contrast'] }` to see just contrast.
- Expected suspicious pairs: `--color-kerf-text-tertiary (#685D52)` on `--color-kerf-bg-base (#181410)` at body copy sizes — check ratio, likely passes.
- Any fails at small sizes: switch to `--color-kerf-text-secondary (#9A8E80)` or bump font-size. Design tokens stay unchanged; only usage changes.
- Document the final pass/fail ratio table for all five text tokens against the three bg tokens in `docs/a11y.md`.

## Risks

- **Unknown violation count.** If axe surfaces more than ~20 critical + serious violations, this PR is genuinely too large for a single clean diff. Mitigation: after Wave 1 baseline, pause and report the count. If alarming, split into a Wave-2 PR and a Wave-3 PR.
- **Recharts a11y gaps.** The dashboard renders several charts. Axe may flag them for missing titles/descriptions. Fix the easy ones (wrap in labelled regions, add title+desc via props where Recharts supports it); defer deep fixes if it becomes its own subproject.
- **Global `:focus-visible`** — may visually regress a few places that styled focus differently. Spot-fix per component.
- **Skip link placement** — must be the first focusable element in the tab order. Verify it renders before `AppNav` in the DOM.
- **Playwright install friction** — `npx playwright install` downloads browsers (~200MB). First-time developer setup gets longer. Documented in `docs/a11y.md` and `README.md` prerequisites.
