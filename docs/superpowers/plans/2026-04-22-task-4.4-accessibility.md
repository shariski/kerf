## Task 2: Author the a11y sweep spec + run baseline

**Files:**

- [ ] **Step 1: Ensure the dev server is running and you have a seeded dev DB**

- [ ] **Step 2: Create the a11y spec**

Create `tests/a11y.spec.ts`:

```ts
import { test, expect } from '@playwright/test';
import AxeBuilder from '@axe-core/playwright';

const WCAG_TAGS = ['wcag2a', 'wcag2aa', 'wcag21a', 'wcag21aa'];

/**
 * Axe sweep over every route/state reachable without typing a live
 * session. States that require mid-typing interaction (active typing,
 * post-session summary) are excluded per docs/a11y.md known deferrals.
 */

const DESKTOP_VIEWPORT = { width: 1280, height: 800 };
const MOBILE_VIEWPORT = { width: 375, height: 812 };

type PageCase = {
  name: string;
  path: string;
  viewport?: { width: number; height: number };
  setup?: (page: import('@playwright/test').Page) => Promise<void>;
};

const pageCases: PageCase[] = [
  { name: 'home (returning user)', path: '/' },
  { name: 'login', path: '/login' },
  { name: 'onboarding', path: '/onboarding' },
  { name: 'practice pre-session', path: '/practice' },
  { name: 'practice drill pre-drill', path: '/practice/drill' },
  { name: 'dashboard', path: '/dashboard' },
  { name: 'keyboards', path: '/keyboards' },
  { name: 'mobile gate (home)', path: '/', viewport: MOBILE_VIEWPORT },
  {
    name: 'practice pause overlay',
    path: '/practice',
    setup: async (page) => {
      // Trigger the pause overlay via Esc. The page must have focus; click the CTA container first.
      await page.locator('.kerf-pre-session').click().catch(() => {});
      await page.keyboard.press('Escape');
    },
  },
];

for (const tc of pageCases) {
  test(`no critical or serious a11y violations: ${tc.name}`, async ({ page }) => {
    await page.setViewportSize(tc.viewport ?? DESKTOP_VIEWPORT);
    await page.goto(tc.path);
    await page.waitForLoadState('networkidle');
    if (tc.setup) await tc.setup(page);

    const results = await new AxeBuilder({ page })
      .withTags(WCAG_TAGS)
      .analyze();

    const critical = results.violations.filter((v) => v.impact === 'critical');
    const serious = results.violations.filter((v) => v.impact === 'serious');

    if (critical.length || serious.length) {
      // Print a readable digest for the engineer fixing the issues.
      console.log(
        `\n[${tc.name}] violations:\n` +
          [...critical, ...serious]
            .map(
              (v) =>
                `  [${v.impact}] ${v.id}: ${v.description}\n    nodes: ${v.nodes.length}`,
            )
            .join('\n'),
      );
    }

    expect(critical, `critical violations on ${tc.name}`).toEqual([]);
    expect(serious, `serious violations on ${tc.name}`).toEqual([]);
  });
}
```

# kerf — accessibility contract

**Target:** WCAG 2.1 AA.

## How to verify

Prerequisites (one-time per developer):

\`\`\`bash
npx playwright install chromium
\`\`\`

Then:

\`\`\`bash
npm run test:a11y
\`\`\`

This starts (or reuses) the dev server on port 3000 and runs `axe-core`
over every non-typing-session route + state, asserting zero critical
+ zero serious violations.

## What's covered

| Route / state | Viewport |
|---------------|----------|
| `/` (returning user) | 1280×800 |
| `/` (mobile gate) | 375×812 |
| `/login` | 1280×800 |
| `/onboarding` step 1 | 1280×800 |
| `/practice` pre-session | 1280×800 |
| `/practice` pause overlay | 1280×800 |
| `/practice/drill` pre-drill | 1280×800 |
| `/dashboard` | 1280×800 |
| `/keyboards` | 1280×800 |

## What's not covered (known deferrals)

- **Active-typing state.** Hard to automate — requires mid-session
  keystrokes. Manually verified: every interactive control has a
  focus ring, Esc opens pause overlay.
- **Post-session summary.** Requires completing a session first.
- **Real screen-reader QA** (NVDA / JAWS / VoiceOver). Post-beta.
- **Recharts chart data points.** The chart regions are labelled,
  but keyboard-navigable data-point exploration is not implemented.

## Keyboard reference

### Global
- **Tab** — focus next interactive element
- **Shift+Tab** — focus previous interactive element
- **Tab (on fresh page)** — skip-to-main-content link slides in; Enter jumps to main
- **Esc** — close current overlay / dismiss banner

### Dashboard
- **j** — scroll down
- **k** — scroll up
- **gg** — jump to top
- **G** — jump to bottom

### Practice session
- **Enter** — start adaptive session (pre-session)
- **Tab** — restart exercise during active typing
- **Esc** — open pause overlay
- **Enter** (in pause) — resume
- **Tab+Enter** (in pause) — restart

### Keyboard switcher (dashboard)
- **Enter / Space** — open dropdown
- **↓ / ↑** — move focus between profile items
- **Enter** — select profile
- **Esc** — close dropdown + return focus

## Contrast

All text / background combinations used in the design system pass
WCAG AA (4.5:1 for small text, 3:1 for large text) against the
dark-espresso base (`#181410`).

## prefers-reduced-motion

All animations longer than 200ms are guarded by
`@media (prefers-reduced-motion: reduce) { transition-duration: 1ms; }`.
Coverage verified by `grep -c "prefers-reduced-motion" src/styles.css`
— currently 13 blocks.
