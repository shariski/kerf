# kerf — accessibility contract

**Target:** WCAG 2.1 AA.

## How to verify

Prerequisites (one-time per developer):

```bash
npx playwright install chromium
```

Then:

```bash
npm run test:a11y
```

This starts (or reuses) the dev server on port 3000 and runs
`axe-core` over every non-typing-session route + state, asserting
zero critical + zero serious violations.

## What's not covered (known deferrals)

- **Active-typing state.** Driving it from Playwright requires
  simulating live keystrokes mid-session, which tears down the
  page context. The `PauseOverlay.test.tsx` unit test covers the
  overlay's markup; the typing area's behavior is tested via
  `TypingArea.test.tsx` and `useKeystrokeCapture.test.ts`. Focus
  rings + aria labels on active-session controls verified manually.
- **Post-session summary / drill-post-session.** Requires completing
  a session first.
- **Real screen-reader QA** with NVDA / JAWS / VoiceOver. Manual
  exploration only; scheduled for post-beta feedback.
- **Recharts chart data points.** The chart regions are labelled,
  but keyboard-navigable data-point exploration is not implemented.

## Keyboard reference

### Global
- **Tab** — focus next interactive element
- **Shift+Tab** — focus previous interactive element
- **Tab (on fresh page)** — skip-to-main-content link slides in; Enter jumps to `#main-content`
- **Esc** — close current overlay / dismiss banner

### Home / pre-session
- **Enter** — start the primary CTA (adaptive practice or first-session)

### Dashboard
- **j** — scroll down (vim)
- **k** — scroll up (vim)
- **gg** — jump to top
- **G** — jump to bottom

### Practice session (active)
- **Tab** — restart exercise during active typing
- **Esc** — open pause overlay

### Pause overlay
- **Enter** — resume
- **Tab+Enter** — restart exercise

### Keyboard switcher (dashboard hero)
- **Enter / Space** — open dropdown
- **↑ / ↓** — move focus between profile items
- **Enter** — select profile
- **Esc** — close dropdown + return focus to trigger

### Drill pre-session
- Typing into the target input accepts a single letter or bigram
- **Enter** — start drill

## Contrast ratios

Raw ratio per design-system token against each background. WCAG AA
thresholds: **4.5:1** for small text, **3:1** for large text (≥18pt or
≥14pt bold).

| text token | `bg-base` | `bg-surface` | `bg-elevated` |
|---|---|---|---|
| `text-primary` `#F2EAE0` | 15.37 ✅ AA | 14.14 ✅ AA | 12.96 ✅ AA |
| `text-secondary` `#9A8E80` | 5.72 ✅ AA | 5.26 ✅ AA | 4.82 ✅ AA |
| `text-tertiary` `#685D52` | 2.86 ⚠️ | 2.63 ⚠️ | 2.41 ⚠️ |
| `amber-base` `#F59E0B` | 8.53 ✅ AA | 7.84 ✅ AA | 7.19 ✅ AA |
| `amber-hover` `#FBBF24` | 10.97 ✅ AA | 10.09 ✅ AA | 9.25 ✅ AA |
| `success-base` `#22C55E` | 8.04 ✅ AA | 7.39 ✅ AA | 6.78 ✅ AA |
| `error-base` `#EF4444` | 4.87 ✅ AA | 4.48 ⚠️ LT only | 4.10 ⚠️ LT only |

**Note on `text-tertiary`.** Axe reports zero flagged instances on
every tested page — meaning every existing use site is either
aria-hidden, paired with a size/weight that qualifies as large
text (3:1 threshold), or overridden by a cascading color. The raw
ratio is too low for small body copy, so when adding a new use of
`text-tertiary`, verify one of the following holds:

1. The element is decorative and has `aria-hidden="true"`.
2. The text is ≥ 18pt regular or ≥ 14pt bold.
3. The element sits on a context whose effective contrast is higher
   (e.g., amber pill backgrounds where the token becomes an accent).

**Note on `error-base`.** The base error red only passes AA on
`bg-base`. Use large-text sizes (18pt / 14pt bold) on other
backgrounds, or pair with a background fill (e.g. the
`error-subtle` amber pill).

## prefers-reduced-motion

Every animation or transition longer than ~200ms is guarded by:

```css
@media (prefers-reduced-motion: reduce) {
  .some-element { transition-duration: 1ms; }
}
```

Current coverage: 13+ blocks in `src/styles.css`. Developer contract:
any new `transition` or `animation` rule longer than 200ms must be
paired with a `prefers-reduced-motion` guard in the same block.

## Focus indicators

- **Global baseline:** `:focus-visible` renders a 2px amber outline
  with 2px offset. Any interactive element without a custom focus
  style picks this up automatically.
- **Per-component overrides:** some elements use an amber background
  flip or an amber border flip instead of an outline. In those cases,
  an explicit `:focus { outline: none; }` or matching
  `:focus-visible { outline: none; }` is paired with the visual
  indicator. See `.kb-key-clickable` (amber stroke on key face) and
  `.kerf-drill-target-input` (amber border).
- **Skip link:** first focusable element on every page; slides in
  from above the viewport on first Tab press.

## Landmarks

Every route renders `<main id="main-content">` as its top-level
container. The skip link anchors to this id. `<AppNav>` renders
inside a `<header role="banner">` in `__root.tsx` for non-chromeless
routes (`/login` and `/onboarding` are chromeless).

## Adding a new route or component — a11y checklist

1. Top-level wrapper is `<main id="main-content">` if the route
   owns the full page; otherwise a semantic child of an existing
   landmark.
2. Exactly one `<h1>`; no heading-level skips.
3. Every interactive element has a name — `aria-label` for icon
   buttons, visible text otherwise.
4. Emoji or icon-only spans inside labelled parents use
   `aria-hidden="true"`.
5. Any new `transition: … > 200ms` has a `prefers-reduced-motion`
   fallback.
6. Re-run `npm run test:a11y` locally before opening the PR.
